GoalFitApp Test Evidence

Overview

This website contains the functional and non-functional test evidence for the GoalFitApp solution. Each requirement has one or more test cases, with each test case having an associated video.

Tests are grouped by priority level:

  • Must-Have: Core functionality required for the app to meet its primary user stories
  • Should-Have: Useful enhancements to be implemented if time allows that would improve the user experience
  • Could-Have: Optional features that add additional functionality
  • Non-Functional Requirements (NFRs): Performance, accuracy, security, and responsiveness tests
