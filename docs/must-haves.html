Must-Have Functional Tests

Summary Table

User Story Tests
M1 — As a cyclist, I want to be able to create a goal so that I have something to work towards. M1.1, M1.2
M2 — As a cyclist, I want to be given mini goals for a goal I have created so that I know where to start and have an achievable plan. M2.1, M2.2
M3 — As a cyclist, I want the system to determine an informed baseline / starting point for my mini-goals, so that the range of distances and elevations within the mini-goals are personalised to me. M3.1, M3.2
M4 — As a cyclist, I want to be able to record workouts so that I know when goals / mini-goals are completed and I know to move to the next mini-goal. M4.1, M4.2, M4.3, M4.4
M5 — As a cyclist, I want to see my progress visually so that I can see how I am improving and progressing toward achieving my main goal. M5.1, M5.2
M6 — As a user, I want to be able to register and log in so that my goals and workouts are saved to my account. M6.1, M6.2, M6.3, M6.4, M6.5, M6.6

Goal Creation Tests

M1.1 — Create goal with valid input → mini-goals created

M1.2 — Create goal with invalid input → validation prevents save (no mini-goals created)

M2.1 — Create Goal generates mini-goals with user workout history

M2.2 — Create Goal generates mini-goals using fallback of global workout history

M3.1 — Mini-goal baseline is based on the user's highest past workout

M3.2 — Mini-goal baseline uses fallback based on user’s Ability Level

Workout Logging Tests

M4.1 — Add workout (valid input) with no goal completion

M4.2 — Add workout (valid input) which completes a mini-goal (but not main goal)

M4.3 — Add workout (valid input) which completes the main goal

M4.4 — Add workout with invalid input — validation prevents save

Dashboard Display Tests

M5.1 — Dashboard shows progress bar and updates after mini-goal completion

M5.2 — Dashboard shows weekly and all-time stats correctly

Login / Registration and Security Tests

M6.1 — Register with a valid email/password → account created

M6.2 — Register with duplicate email → error shown, account not created

M6.3 — Log in with correct credentials → dashboard shown

M6.4 — Log in with wrong credentials → error shown

M6.5 — View dashboard without being logged in → user invited to login or register

M6.6 — Visit a page that requires authentication when not logged on → user automatically redirected to login page (requested page not shown)
